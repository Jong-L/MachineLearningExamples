让我用具体的例子来解释为什么折扣因子为 1 时，增大 `episode_length` 就能得到正确结果。

## 核心问题：有限截断 vs 无限 horizon

当折扣因子 γ=1 时，真实的累积回报应该是：
$$G_t = R_{t+1} + R_{t+2} + R_{t+3} + ... = \sum_{k=0}^{\infty} R_{t+k+1}$$

但代码中实际计算的是：
$$\hat{G}_t = R_{t+1} + R_{t+2} + ... + R_{t+T} = \sum_{k=0}^{T-1} R_{t+k+1}$$

其中 T = `episode_length`

## 具体示例分析

让我用一个简化的例子来说明：

### 场景设定
假设从状态 (4,2) 到目标状态 (3,2) 的路径如下：
- (4,2) → (3,2) [目标，奖励 +1] → (3,2) [原地不动，奖励 +1] → (3,2) → ...

### 情况 1：episode_length = 5（太短）

**从 (4,2) 向下走到 (3,2)**：
```
步数 0: 在 (4,2)，执行"下"动作
步数 1: 到达 (3,2) [目标]，获得奖励 +1
步数 2: 按策略随机选择，可能向右走到 (3,3)，获得奖励 0
步数 3: 继续随机游走，可能进入禁止区域 (3,1)，获得奖励 -1
步数 4: 离开禁止区域，获得奖励 0
步数 5: episode 结束

总回报 = 1 + 0 + (-1) + 0 + 0 = 0
```

**从 (3,2) 目标状态原地不动**：
```
步数 0: 在 (3,2)，执行"原地不动"
步数 1: 仍在 (3,2)，获得奖励 +1
步数 2: 按策略随机选择，可能向上走到 (2,2)，获得奖励 0
步数 3: 继续随机游走...
步数 4: ...
步数 5: episode 结束

假设路径：(3,2)→(2,2)→(1,2)→(1,1)[禁]→(1,2)
总回报 = 1 + 0 + 0 + (-1) + 0 = 0
```

**问题**：由于采样太短，两种情况的估计都很不准确！

### 情况 2：episode_length = 1000（足够长）

**从 (4,2) 向下走到 (3,2)**：
```
前几步：(4,2)→(3,2)[+1]→随机游走...
中间：可能会多次回到目标状态 (3,2)
后期：在目标状态停留较长时间（如果策略允许原地不动）

假设在 1000 步内：
- 访问目标状态 200 次，每次 +1
- 访问禁止区域 50 次，每次 -1
- 其他区域奖励为 0

总回报 ≈ 200×1 + 50×(-1) = 150
平均回报 ≈ 150/1000 = 0.15（每步期望）
```

**从 (3,2) 目标状态开始**：
```
如果在目标状态选择"原地不动"：
- 可以持续获得 +1 奖励
- 1000 步都停留在目标状态

总回报 = 1000×1 = 1000
平均回报 = 1000/1000 = 1.0（每步期望）
```

**或者如果策略是随机游走**：
```
在 1000 步的长轨迹中：
- 会多次访问目标状态（因为策略是均匀分布）
- 根据马尔可夫链的平稳分布，会以一定概率停留在各个状态
- 长期来看，目标状态的访问频率最高（因为一旦到达就会获得高奖励）

总回报可能 = 300×1 + 80×(-1) = 220
平均回报 ≈ 0.22
```

## 关键差异

### 当 γ=1 且 episode_length 较短时：

| 状态 | 真实值（无限步） | 估计值（50 步） | 误差原因 |
|------|----------------|---------------|----------|
| (3,2) 目标 | 很高（可持续获得 +1） | 可能被低估 | 采样中离开了目标区域 |
| (4,2) 邻近 | 较高（容易到达目标） | 可能波动大 | 有时能到目标，有时不能 |

### 当 γ=1 且 episode_length 很长时：

| 状态 | 真实值 | 估计值（1000 步） | 准确性 |
|------|--------|----------------|--------|
| (3,2) 目标 | 很高 | 接近真实值 ✓ | 充分采样 |
| (4,2) 邻近 | 较高 | 接近真实值 ✓ | 充分采样 |

## 数学解释

对于 γ=1 的情况，状态值的定义是：
$$V(s) = E_\pi[\sum_{t=0}^{\infty} R_t | S_0 = s]$$

这是一个**无限和**。蒙特卡洛方法用有限和近似：
$$\hat{V}(s) = \frac{1}{N}\sum_{i=1}^{N}\sum_{t=0}^{T-1} R_t^{(i)}$$

**截断误差**：
$$\text{Error} = E[\sum_{t=T}^{\infty} R_t]$$

- 当 T 很小时，这个误差很大
- 当 T→∞ 时，误差→0

## 为什么 γ<1 时不需要很长的 episode_length？

当 γ<1（例如 γ=0.9）时：
$$G_t = R_{t+1} + \gamma R_{t+2} + \gamma^2 R_{t+3} + ...$$

远期奖励被指数级衰减：
- γ^10 ≈ 0.35（10 步后的奖励只值原来的 35%）
- γ^50 ≈ 0.005（50 步后的奖励几乎可以忽略）

所以即使 `episode_length` 较短，也能得到很好的近似！
